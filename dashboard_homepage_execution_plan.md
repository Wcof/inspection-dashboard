# Dashboard 首页整改计划（执行版）

## 1. 目标
- 首页回归“总览端”：看态势、看风险、看状态、看播报，不做重执行。
- 信息结构清晰：5 秒内看懂“当前状态 + 风险 + 入口”。
- 地图为主舞台：名称/图标/路径开关明确联动。

## 2. 信息架构整改（P0）

### 2.1 总体运营概览重组
- 去掉“待机机器人数”“当日异常数”。
- 新口径：机器人总数、任务中、充电中、当日巡检任务数、机器人运行时间。
- 右上角“实时监控”改“运行总时长（所有机器人汇总）”。

### 2.2 巡检块融合
- 巡检点概览并入巡检总览（子块样式，弱化独立大标题）。
- “已完成巡检任务数”改“巡检任务总次数”。

### 2.3 机器人总览口径
- 状态三类：执行中 / 返航中 / 充电中。
- 指标调整：保留平均续航；“剩余里程”改“平均时速”；“今日总里程”改“当前里程 + 当日里程”。

### 2.4 风险颜色策略
- 正常状态不使用红色；仅异常时才用红/橙。
- 默认使用中性色（与“气体异常”一致的色系）表达安全风险总览。

### 2.5 门禁指标概览新增
- 展示“机器 / 远端”灰字提示。
- 指标值格式统一：`1/0`、`1/1`。

### 2.6 充电桩概览重做
- “当前服务机器人数”改“充电总次数”。
- 保留“闲置数量”。
- 移除“使用率”“空闲/服务/排队/异常”组合块。
- Mock 扩充到 6 条充电站数据。

## 3. 地图工具栏与联动规则（P0）

### 3.1 map-toolbar 中文命名
- UI 文案改为中文名（建议“地图操作栏”）。

### 3.2 工具项文案改名
- `标签显示` -> `名称显示`
- `图层显示` -> `机器人显示`
- `巡检点状态` -> `巡检点显示`
- `机器人轨迹` -> `巡检路径`

### 3.3 开关联动规则（强约束）
- 名称显示：只控制“巡检点/机器人 名称+状态文字”；且仅在对应图标已开启时生效。
- 机器人显示：只控制机器人 icon 显示/隐藏。
- 巡检点显示：只控制巡检点 icon 显示/隐藏。
- 巡检路径：控制路径显示；并与“机器人显示/巡检点显示”联动（任一关闭时路径受限或隐藏）。

## 4. 播报与动态感（P0）
- 信息播报固定在右侧“充电站概览”下方。
- 告警置顶固定不动。
- 普通消息采用“自下而上”纵向轮播（不左右滚动）。

## 5. 交互边界（P0）
- 首页只保留轻交互：点选查看、开关切换、入口跳转。
- 不新增调度闭环、冲突决策、复杂控制流程。

## 6. 实施步骤
1. 改 `index.html` 结构和文案（模块顺序、标题、字段容器）。
2. 改 `main.css`（层级、卡片密度、颜色策略、播报动画）。
3. 改 `app.js`（聚合口径、状态字典、工具栏联动、播报数据源）。
4. 本地验收：`node --check app.js` + `npm run build:pages`。
5. 人工验收后再合并到 `main`。

## 7. 验收清单
- 总体运营概览字段按新口径替换完成。
- 巡检点概览并入巡检总览完成。
- 机器人总览三状态 + 新指标口径生效。
- 风险颜色策略符合“谨慎用红橙绿”。
- 门禁指标概览显示“机器/远端”与 `1/0` 格式。
- 充电桩概览字段与 6 条 Mock 数据生效。
- 地图工具栏中文命名与 4 项联动规则生效。
- 播报“告警置顶 + 纵向轮播”生效。

## 8. 交付说明
- 文件用途：供其他 agent 直接对照执行与打勾验收。
- 本文件仅定义首页（dashboard）整改，不扩展调度台/驾驶舱/APP 页面。
