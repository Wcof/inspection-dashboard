# Dashboard 布局优化更新

## 更新内容

### 1. 布局优化
- **去除了顶部统计卡片区域**，保持总体左右布局
- **调整了面板高度**：将面板高度改为自适应，从顶部导航栏下方一直延伸到底部
  - 原布局：固定高度（top: 120px, bottom: 120px）
  - 新布局：自适应高度（top: 16px, bottom: 20px）
  - 优化了顶部间距，使左右面板与顶部导航栏更加对齐
- **优化了面板内边距**：将面板内边距从12px调整为16px，提升视觉效果
- **调整了main-container样式**：移除了顶部padding，避免影响面板定位

### 2. 指标重组
- **左侧面板**：添加了"领导关注指标"卡片
  - 包含：机器人总数、执行任务数、任务进度、当日里程
- **右侧面板**：添加了"运营状态指标"卡片
  - 包含：当前充电数、当日告警

### 3. 交互方案优化
- **新增告警弹窗**：点击"巡检点覆盖与异常"中的警告点和严重点时，不再切换面板，而是弹出模态窗口显示告警详情
- **弹窗内容**：包含告警时间、级别、状态、缺陷详情、证据图片等信息
- **交互功能**：支持确认异常、解除误报等操作
- **视觉效果**：使用半透明背景和模糊效果，提升用户体验

### 4. 样式调整
- **添加了stat-card.small样式**：使统计卡片在侧边栏中显示更紧凑
  - 调整了图标大小、字体大小和内边距
- **保持了原有的深色主题**：确保视觉风格一致性

### 5. 数据保留
- 所有原有的数据和渲染逻辑保持不变
- 只是调整了显示位置和交互方式

### 6. 元素移除
- **移除了右侧面板中的Rich List块**：从"巡检点覆盖与异常"部分移除了风险列表HTML元素
- **数据保持完整**：相关数据结构和渲染逻辑保持不变，只是移除了显示部分

### 7. 领导视角优化
- **顶部KPI栏优化**：
  - 从技术指标改为领导关注的宏观指标
  - 运营状态、任务完成率、设备完好率、风险闭环率
  - 增加卡片式设计，提升视觉层次感
- **左侧面板重构**：
  - 将"领导关注指标"改为"总体运营概览"
  - 优化指标展示：机器人总数、任务总量、运营效率、安全指数
  - 增加图标大小和视觉效果
- **右侧面板重构**：
  - 将"运营状态指标"改为"安全风险指标"
  - 重点突出：当日告警、严重问题、平均响应时间
  - 强化安全风险监控视角
- **样式全面升级**：
  - 提升kpi-item样式，增加背景和边框效果
  - 优化stat-card.small样式，增加hover效果和视觉层次
  - 改进summary-item样式，统一使用金色主题色
  - 增加过渡动画，提升交互体验
  - **总体运营概览**：重新设计为现代化的kpi-card布局，采用2x2网格，增加渐变背景和hover效果
  - **安全风险指标**：重新设计为垂直排列的kpi-card布局，每个卡片使用不同的颜色主题以突出风险等级
  - **总体运营概览优化**：
    - 机器人总数 → 机器总数
    - 任务总量 → 当日任务总数
    - 运营效率 → 当日任务进度
    - 安全指数 → 已排查巡检设备总数
    - 新增：机器总运行时长
    - 去除：平均电量
    - 指标合并：机器总数与总运行时长合并，当日任务总数与任务进度合并
    - 调整为1x3网格布局，更紧凑地展示3个卡片
  - **安全风险指标优化**：
    - 去除：平均响应时间、风险闭环率
    - 优化布局：当日告警总数与分级统计（严重/一般/可忽略）融合在一个紧凑卡片中
    - 调整为单行布局，大幅节省空间
    - 视觉设计：使用更紧凑的字体和间距，保持信息清晰
- **指标体系优化**：
  - 新增运营效率指标（综合利用率、完成率、覆盖率）
  - 新增安全指数（基于告警和故障计算）
  - 新增设备完好率和风险闭环率
  - 优化数据可视化，突出关键指标
